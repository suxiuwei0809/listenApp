# 健身APP开发计划

## 一、需求分析

### 核心功能需求

| 需求编号 | 功能描述 | 需求来源 |
| :--- | :--- | :--- |
| R001 | **运动记录**：记录每日运动类型、时长、消耗卡路里 | 健身基础需求 |
| R002 | **训练计划**：提供多种训练计划（如减脂、增肌、塑形） | 系统化健身指导 |
| R003 | **动作库**：包含各种健身动作的图文/视频教程 | 学习正确动作 |
| R004 | **进度追踪**：记录体重、体脂率等身体数据变化 | 可视化健身成果 |
| R005 | **打卡提醒**：设置运动提醒，养成健身习惯 | 习惯养成辅助 |
| R006 | **计步功能**：获取手机传感器数据，统计每日步数 | "拿手机的一些数据，比如说每天走了多少步" |
| R007 | **平板支撑鼓励**：专门的平板支撑计时器，定时给予鼓励 | "平板支撑的鼓励模块，坚持不下去的时候给我鼓励" |
| R008 | **俯卧撑计数**：支持手动计数，记录每组次数和组数 | "增加俯卧撑计数" |
| R009 | **腹肌轮计数**：支持手动计数，记录每次训练的次数 | "增加腹肌轮计数" |
| R010 | **健身房训练计划**：首页展示开练计划，支持选择不同训练目标（练背/练胸/练腿/练腹肌/减脂），提供对应训练流程（热身→主训练→拉伸），支持按步骤执行 | "去健身房第一步练什么，第二步练什么"，"放在首页，就是开屏页"，"针对不同部位，练背、练胸肌、练腹肌、减脂有不同方案" |

### 非功能需求

| 需求编号 | 描述 |
| :--- | :--- |
| N001 | 仅供个人使用，无需登录注册 |
| N002 | 界面简洁现代，易于操作 |
| N003 | 支持离线使用（数据本地存储） |

## 二、技术方案

### 技术选型

| 分类 | 技术 | 版本 | 选型理由 |
| :--- | :--- | :--- | :--- |
| 语言 | Kotlin | 1.9+ | Android官方推荐，现代语法 |
| UI框架 | Jetpack Compose | 1.6+ | 声明式UI，开发效率高 |
| 架构模式 | MVVM | - | 分离关注点，便于维护 |
| 数据存储 | Room数据库 | 2.6+ | 本地数据持久化 |
| 计步传感器 | Android Sensor API | - | 系统内置计步传感器 |
| 通知 | Android NotificationManager | - | 推送通知提醒 |
| 健康数据 | Google Fit API（可选） | - | 获取更全面的健康数据 |
| 图标 | Material Icons | - | Google官方图标库 |

## 五、关键技术要点

### 1. 数据模型设计

**WorkoutRecord（运动记录）**：
- id: Long
- type: String（运动类型：跑步、力量训练、瑜伽等）
- duration: Int（时长，分钟）
- calories: Int（消耗卡路里）
- date: Long（日期时间戳）

**TrainingPlan（训练计划）**：
- id: Long
- name: String（计划名称）
- description: String（计划描述）
- exercises: List<Exercise>（包含的动作）

**Exercise（动作）**：
- id: Long
- name: String（动作名称）
- category: String（分类：胸部、背部、腿部等）
- sets: Int（组数）
- reps: Int（次数）
- description: String（动作说明）
- tips: String（姿势要点）
- caution: String（注意事项）

**BodyData（身体数据）**：
- id: Long
- weight: Float（体重，kg）
- bodyFat: Float（体脂率，%）
- date: Long（记录日期）

**StepRecord（步数记录）**：
- id: Long
- steps: Int（步数）
- distance: Float（距离，公里）
- calories: Int（消耗卡路里）
- date: Long（日期）

**PlankRecord（平板支撑记录）**：
- id: Long
- duration: Int（时长，秒）
- date: Long（日期时间戳）

**PushupRecord（俯卧撑记录）**：
- id: Long
- totalReps: Int（总次数）
- sets: Int（完成组数）
- maxPerSet: Int（单组最大次数）
- setDetails: List<Int>（每组次数明细）
- date: Long（日期时间戳）

**AbWheelRecord（腹肌轮记录）**：
- id: Long
- totalReps: Int（总次数）
- sets: Int（完成组数）
- maxPerSet: Int（单组最大次数）
- setDetails: List<Int>（每组次数明细）
- date: Long（日期时间戳）

**GymPlan（健身房训练计划）**：
- id: Long
- target: String（训练目标：练背/练胸/练腿/练腹肌/减脂）
- warmup: List<GymStep>（热身环节）
- main: List<GymStep>（主训练环节）
- stretch: List<GymStep>（拉伸环节）

**GymStep（健身房训练步骤）**：
- name: String（动作名称）
- sets: Int（组数）
- reps: String（次数描述，如"12次"或"30秒"）
- tips: String（姿势要点）
- caution: String（注意事项）

**Encouragement（鼓励语）**：
- id: Long
- text: String（鼓励语内容）
- type: String（类型：初期/中期/后期/坚持）

### 需要准备的资源
1. 图标资源（运动类型图标）

### 权限配置
- `android.permission.ACTIVITY_RECOGNITION`（计步传感器权限）
- `android.permission.WAKE_LOCK`（后台计步需要）
- `android.permission.POST_NOTIFICATIONS`（Android 13+通知权限）
